# Archived learnings (epoch:1782951271)

_(consolidated 1 dòng cũ → agents/archive/KNOWLEDGE-1782948170.md)_
- **validated: kitty user `map` silently shadows a DEFAULT binding — always
  cross-check `/usr/share/doc/kitty/kitty.conf`'s commented `kitty_mod+…`
  defaults before claiming a key for a new feature.** `ctrl+shift+minus`
  IS kitty's default zoom-out (`change_font_size all -2.0`); the gsd-style
  3-pane vsplit map stole it with zero warning/conflict-detection from
  kitty itself. Moved vsplit → `ctrl+shift+backslash` (setup.rs:665-668).
  Font-zoom maps apply live on `kitty @ load-config` (unlike window
  decorations, which need a window recreate) — cheaper fix path when
  remote control is on.
- **validated: `~/.omp/capabilities.md` embeds the EXACT tool catalog for every
  registered MCP server, not just a count.** Directly closes the gap that
  caused the earlier codegraph-verb hallucination (`query/deps/callers/defs`
  guessed, none existed — real CLI is `query/callers/callees/impact/…`, see
  entries below). `known_mcp_tool_catalog(server)` in `deploy.rs` is a static
  match on the 4 harness-pinned servers (codebase-memory-mcp/headroom/serena/
  zai-vision) — deliberately NOT spawning each server to introspect live
  (would slow every `8sync harness` run); server names are read live from
  `mcp.json` so an unpinned/user-added server still gets listed (just without
  a catalog, with an explicit "check its own docs" note instead of silently
  omitting it). omp's OWN built-in tools ARE parsed live (from `omp --help`'s
  "Available Tools" block) since that's cheap and always accurate. Wired into
  `APPEND_SYSTEM.md` RULE #0 + the `8sync-recall.ts` hook bundle (survives
  compaction) as "look it up here, never guess".
